# ComfyUI23

一个面向 ComfyUI 的中文友好型仓库，用于收集、演示和实验自定义节点与工作流示例，帮助你更高效地在本地进行 AI 流程的搭建与扩展。

## 特性
- 提供简洁的自定义节点与工作流示例，便于快速上手
- 以中文为主的说明与注释，降低学习成本
- 结构清晰、可扩展，适合继续添加你自己的节点或案例

## 环境要求
- 已安装 ComfyUI（建议使用官方仓库的最新版）
- 推荐 Python 3.10+（按你的 ComfyUI 环境为准）
- 操作系统：Windows（本仓库在 Windows 目录结构下组织）

## 安装与使用
1) 克隆或下载本仓库：

```bash
git clone https://github.com/your-username/ComfyUI23.git
```

2) 将自定义节点放入 ComfyUI 的 `custom_nodes` 目录（如有）：
- 典型路径示例：`ComfyUI/custom_nodes/`
- 重启或启动 ComfyUI，确保节点被正确加载

3) 打开 ComfyUI，加载工作流示例并开始使用
- 可在工作流中查询并使用本仓库提供的节点
- 如遇报错，请参考下文常见问题与排查建议

## 目录结构（示意）
- `README.md`：项目说明与使用指南（当前文件）
- `测试/`：用于试验与演示的文件或脚本（示例目录）

根据你的实际需要，你可以继续添加：
- `nodes/`：自定义节点的 Python 文件
- `workflows/`：工作流 JSON 示例
- `assets/`：图片或静态资源

## 开发指南（自定义节点思路）
- 在 `custom_nodes` 或项目的节点目录中创建 Python 文件
- 编写节点类，定义输入、输出与处理逻辑
- 按 ComfyUI 的节点注册约定进行导出与注册
- 重启 ComfyUI，确认新节点出现在节点列表中

建议：
- 为每个节点写最小可运行示例，便于快速验证
- 在代码中加入中文注释，记录输入/输出的含义

## 常见问题
- 无法加载节点：检查文件是否置于正确的 `custom_nodes` 目录、文件名与类名是否符合约定
- 依赖问题：根据节点功能安装对应依赖（例如 `pip install ...`），并与 ComfyUI 的 Python 环境一致
- 端口占用：启动 ComfyUI 前确认端口未被其他程序占用
